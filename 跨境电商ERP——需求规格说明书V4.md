## 1. 文档概述

### 1.1 编写目的

本说明书是对跨境电商ERP项目进行系统化需求定义的基线文档，明确系统建设目标、业务范围、角色职责、核心业务流程、详细功能需求、数据原则、接口规范和非功能约束。文档作为产品设计、架构设计、数据库建模、接口开发、测试验收与项目管理的核心输入。

### 1.2 适用对象

产品负责人、项目经理、架构师、前后端研发、测试工程师、实施与运维人员，以及运营、采购、仓储、财务等业务负责人。

V4版在V3基础上整合了V11详细设计说明书中已补齐的98项业务能力、PMS集成需求、AI能力分布矩阵，形成更全面、更贴近实际业务操作的需求规格。覆盖从选品开发、销售运营、订单处理、多仓采购备货、仓储作业、物流发货、客服售后到财务核算、数据分析的全链路管理需求，并新增AI能力与PMS集成需求章节。

## 2. 项目背景与建设目标

### 2.1 行业背景

跨境电商卖家面临多渠道（Amazon、Shopify、TikTok Shop、eBay、Walmart等）、多市场、多店铺、多仓库（本地仓、FBA、海外仓、3PL）并存的经营环境。从选品开发、商品刊登、销售运营、订单处理、采购备货、仓储发货、物流跟踪到客服售后和财务核算，各环节数据分散、规则不统一，手工操作效率低、易出错。

### 2.2 痛点分析

- **选品开发**：需求来源多（人工提报、供应商推荐、1688选品、平台采集），产品开发流程不规范，跨部门协作困难，开发效率和出单效果难衡量。
- **刊登运营**：多平台多店铺Listing刊登繁琐，多语种翻译效率低，侵权风险难以自动识别；运营数据分散，广告管理跨平台切换复杂，在线监控和异常预警能力弱。
- **订单处理**：多平台订单需分别登录后台处理，分仓、拆单、合单、物流匹配依赖人工经验，申报和标记发货易出错。
- **多仓备货**：本地仓、FBA、海外仓补货计算依赖excel，缺货与积压并存，采购、收货、质检、入库链条长且协同困难。
- **仓储管理**：库存数据不准，预占、在途、可售口径不统一，质检不良品管理不规范，多仓调拨和盘点效率低。
- **客服售后**：多平台邮件/消息分散，回复质量参差不齐，退款退货处理与库存财务脱节。
- **财务核算**：平台佣金、广告费、物流费、头程、仓储费、退款损耗等成本项多且归集困难，利润核算依赖手工，时效性和准确性不足。
- **数据分析**：各业务节点绩效数据分散，管理层缺乏统一的经营视图。
- **AI能力缺失**：选品、补货、广告优化、风控等场景缺乏AI辅助决策，依赖人工经验判断。

### 2.3 建设目标

建设一套覆盖跨境电商全链路的经营管理ERP平台，形成"选品开发→刊登运营→订单处理→多仓备货→仓储执行→物流发货→客服售后→财务核算→经营分析"的完整闭环，支持多渠道、多店铺、多仓库的一体化管理，实现业务数据实时准确、成本利润清晰可见、运营效率全面提升。同时与PMS（AI选品系统）深度集成，实现AI辅助决策与人工审批的有机结合。

### 2.4 产品定位

本系统定位为跨境经营控制台和业务中台，不是前台商城，不是简单OMS，不是通用财务软件。核心面向平台卖家、独立站、多渠道品牌商，支持混合履约模式（自发货、FBA、海外仓、3PL）。

### 2.5 成功标准

- 覆盖商品、渠道、订单、库存、采购、履约、售后、结算、分析9大核心领域，支持选品开发和广告管理。
- 库存准确率≥99%，订单处理自动化率显著提升。
- 利润偏差≤2%，账单归集自动化覆盖。
- 系统支持至少3个主流销售渠道接入。
- 平台原始数据可长期保留，业务操作全链可审计。
- AI辅助决策覆盖选品、补货、广告、风控4大核心场景。
- PMS集成建议采纳率≥30%，建议执行准确率≥85%。

## 6. 核心业务流程

### 6.1 全链路总览

```
选品开发 → 刊登运营 → 订单处理 → 多仓备货 → 仓储执行 → 物流发货 → 客服售后 → 财务核算 → 经营分析
    ↑                                                                          │
    └──────────────────── PMS AI反馈闭环 ←──────────────────────────────────────┘
```

### 6.2 产品开发流程

```
选品提报/1688选品/出单产品/供应商推荐/PMS AI建议
    → 审核分配 → 产品开发(资料/变体/图片/品牌/UPC/敏感词)
    → 质检标准 → 开发完成 → 上架前配置(限价/标题库/图片库)
    → 刊登上线
```

### 6.3 销售运营流程

```
Listing刊登(价格计算器/标题库/图片库)
    → 运营优化(Buybox采集/调价/跟卖监控/定时上下架/平衡库存)
    → 广告管理(SP/SB/SD/SBV/搜索词提炼/否定关键词/仪表盘/分组分析)
    → 运营监控(销量/价格/Buybox/库存/星级评论异常)
    → Reviews同步 → 质量问题记录 → PDM联动
```

### 6.4 订单处理与发货流程

```
平台订单同步/批量导入
    → 渠道订单(保留原文) → ERP销售单(标准化) → 订单审核(风控/利润)
    → 履约单(分仓/物流优选/物流申报规则/插头国标)
    → 包裹单(配货/打单/拣货/打包/分拣口/发货)
    → 平台标记发货(自动上传追踪号)
    → 运输统计(估算vs实际运费差异)
```

### 6.5 多仓备货与采购流程

```
本地仓备货分析/FBA补货建议(购物车)/海外仓备货分析/PMS AI补货建议
    → 备货计划单 → 采购需求汇总(减去可用库存和采购中数量)
    → 采购单(5种模式：市场/工厂/天猫淘宝/1688/加工)
    → 采购审核(多级审核/自动审核) → 1688下单付款/供应商平台接单
    → 跟单(可视化) → 收货质检 → 销单入库核对 → 付款申请
```

### 6.6 仓储作业流程

```
收货 → 销单(打印进货单/采购条码) → 质检(PC/PDA)
    → 入库(库存事务账记录：在手/预占/可售/在途/不良)
    → 不良品处理(退货/返修/回购换货)
    → 出货质检 → 拣货(PDA) → 打包 → 发货
    → 库存变动(调拨/盘点/手工出入库/Removal订单)
```

### 6.7 售后流程与财务核算

```
邮件/消息(智能分配/假期自动回复/新人审核)
    → 评价管理(中差评/请求评论) → 退换货/补发
    → 退款报告 → 质量问题记录(PDM/WMS联动)
    → 成本归集(8类成本事件) → 利润核算(FIFO)
    → 对账(供应商/物流商) → 平台回款 → 进销存凭证
```

## 7. 详细功能需求

### 7.1 工作台域 (DASHBOARD) ★ \[AI看板]

- **首页数据看板**：工作流通知、采购入库、订单包裹、邮件汇总；各平台销售概况、销售明细、产品/店铺排行榜；常用功能入口、电子日历。
- **常用功能入口**：用户自定义快捷方式、高频功能一键直达、最近访问记录。
- **工作流通知聚合**：审批通知、异常告警、库存预警、订单异常、授权失效等多类型工作流通知统一聚合展示，按紧急程度分级。
- **帮助中心**：视频教程、功能帮助指引、错误提示搜索QA、公告栏。
- **个人中心**：头像密码修改、KPI考核相关功能（仓库人员KPI、开发人员KPI等）。
- **电子日历**：运营日历、任务日历、促销日历、备货日历。
- 首页数据均受权限控制。

### 7.2 组织权限域 (IAM)

**功能要求**：

- 公司管理：部门、组织架构、角色、岗位、操作人员账号管理。
- 岗位管理：岗位定义、岗位层级、岗位与部门关联、岗位人员分配。
- 角色权限：支持菜单权限+数据权限（按组织、店铺、渠道、仓库等10维隔离），支持产品/Listing/广告等对象级别权限。
- 对象级权限：用户自定义产品角色（数据权限），可查看全部或仅与自己相关的产品；支持产品、Listing、广告等业务对象的精细权限控制。
- 登录操作日志：记录所有用户的登录和操作，支持审计。
- 系统参数配置：大量业务开关（如采购下单必须为审核供应商、采购人员只能生成自己负责的采购单、检测订单利润率等）。

### 7.3 产品开发域 (PDM) ★ \[AI选品]

**选品开发来源**：

- 选品提报：开发/销售人员线上提报，支持审核、分配或认领开发任务，生成系统SKU。
- 产品采集：通过链接采集Amazon、速卖通、wish、eBay、Lazada、阿里国际、1688、淘宝、天猫、拼多多等平台信息转为新品。
- 出单产品：先上架销售、出单后再关联或生成SKU，再采购发货。
- 1688选品＆铺货：对接1688精选货源（48小时发货率、好评、价格优势），一键铺货至系统铺货列表，再生成SKU。
- 供应商平台新品：供应商在供应商平台推送新品供卖家选品。
- PMS AI选品建议：接收PMS选品评分、竞品分析、选品建议，自动创建选品提报。

**产品开发管理**：

- 产品资料增删改查、类别管理。
- 支持加工、组合(Bundle)、变体（母体/子体管理），批量编辑多变体（如服装颜色尺码一键生成）。
- 批量上传图片、图片空间管理。
- 产品开发审核，批量分配开发、采购、编辑、美工人员，统计各阶段产品数量。
- 用户自定义产品角色（数据权限），可查看全部或与自己相关的产品。
- 运营管理：渠道SKU管理、组合产品管理、产品销量活跃度规则（爆款/滞销款）。

**产品资料维护**：

- 渠道SKU映射：绑定各店铺Seller SKU与内部SKU，支持自动生成规则。
- 分类质检：按产品类别设置质检标准，质检环节自动展示。
- 自定义物流属性：用于订单审核规则匹配运输方式。
- 物流包材：维护包材成本，发货后自动计入订单利润。
- 产品问题记录：汇集质检、客服售后、评论中记录的质量问题，支持分析优化。

**上架前配置**：

- 平台产品限价：防止多店铺同站点内部价格战。
- 知识产权管理：流程化侵权记录和查询。
- UPC管理：导入UPC，刊登时自动调用未使用码。
- 敏感词库：维护敏感词，文案编辑或刊登时自动提醒。
- 品牌管理：维护品牌，刊登时快速选择。
- 标题库/图片库：维护标题库和图片库，刊登时系统自动随机调用。

**开发人员KPI**：

- 开发人员开发SKU数、开发效率、产品出单率、销售总利润。
- KPI管理和提成计算。

### 7.4 销售运营域 (SOM)

**高效刊登**：

- 支持Amazon、eBay、速卖通、Wish、Walmart、Mercado Libre、Shopee等多平台Listing刊登管理。
- 多店铺多站点单个/批量/复制刊登；支持一键翻译文案快速刊登多语种站点。
- 刊登时自动调用标题库/图片库随机匹配标题图片，减少关联风险。
- 使用Listing价格计算器自动计算售价，确保利润。
- 统计刊登效率和出单效果报表。

**Listing管理与优化**：

- 权限分明：仅能查看和操作自己相关的Listing，部门负责人可看下级。
- 批量调价、促销、修改可售数；平衡库存策略自动补货。
- 为滞销品设置降价清库计划。
- 批量修改多语种文案和图片。
- 采集buybox信息，自动调价提升buybox获得率。
- 跟卖监控与提醒。
- 自动定时上下架、手动批量下架/删除。
- 通过Reviews发现产品问题，记录质量投诉归类。
- 多平台账号健康安全状态同步。
- 销售小组设置，分组管理统计业绩。

**运营监控**：

- 设置销量、价格、图片/标题、Buybox、在售状态、FBA库龄库存、星级评论等监控指标，异常时通过短信、邮件、钉钉提醒。
- 运营日历：记录商品/店铺运营事件，便于复盘。

### 7.5 广告管理域 (ADS) ★ \[AI优化]

**广告活动管理**：

- 多平台广告集成（Amazon、eBay、Wish）；已深度对接Amazon广告。
- 创建SP、SB、SD、SBV四种广告类型，同步后台广告数据。
- 管理人员和运营人员双风格界面。
- 统一批量管理多店铺广告活动，查看ACoS、曝光、点击、转化、花费等。
- 批量更改广告状态、预算、竞价。

**关键词管理**：

- 用户搜索词管理：将优质词添加为关键词，设为否定关键词。
- 搜索词提炼：自动发现高转化搜索词，从用户搜索词报告中提炼优质搜索词添加到广告活动。
- 否定关键词管理：避免无效点击。

**广告分析与策略**：

- 仪表盘：可视化不同广告表现，排名最佳/最差店铺、高效/低效关键词，支持按活动/组/关键词/ASIN维度。
- 分组分析：自由组合广告活动/组/商品分析。
- 广告策略：自动调价（基于曝光、点击等条件设置规则调整竞价/预算）。
- 报告导出。

**PMS广告优化集成**：

- 接收PMS广告策略推荐、自动调价指令、关键词建议。
- 支持PMS操作回滚。
- PMS优化开关控制。

### 7.6 订单域 (OMS) ★ \[AI风控]

**订单接入**：

- 平台授权后通过API自动下载订单；未对接平台支持批量导入订单。
- 订单来源渠道添加店铺账号及授权。

**订单三层分离**：

- 渠道订单：保留平台原始数据，不做翻译。
- ERP销售单：统一各平台差异，提供标准视图。
- 履约单：面向仓库执行，包含配货/拣货/打包。
- 包裹单：面向物流追踪，包含承运商/追踪号/重量。

**订单处理自动化**：

- 订单审核策略与分仓物流优选：自动匹配最优发货仓库和物流渠道，识别异常订单设为待审核。
- 物流申报规则：按国家、物流渠道、品类、重量等设置申报规则，结合预生成策略自动申报获取追踪号。
- 平台标记发货规则：自动上传追踪号，平台状态自动转为已发货。
- 风险订单检测：自动识别恶意买家重复下单，加入黑名单的用户订单自动拦截。
- 插头国标规则：按收货国家自动匹配对应插头规格SKU。
- 自定义发票设置：配置打印模板。
- 订单拆单/合单/有货先发等操作。

**发货执行**：

- 海外仓发货：订单自动或手动同步至海外仓，由海外仓发货并返回状态和追踪号。
- 本地仓发货：
  - 调库配货：为订单锁定库存生成包裹；支持自动调库配货规则。
  - 客户订单包裹：支持默认打单、单品单件、单品多件、多品多件扫码打单。
  - 拣货：A4拣货单或PDA电子拣货任务，支持自动生批规则。
  - WMS客户端：扫描出单、称重发货，可重新打印面单。
- 分拣口设置：配合智能分拣硬件自动按物流渠道分拣。
- 运输统计：自动统计发货包裹数和运费，生成交运批次，比照估算运费与实际运费差异。
- 交运批次管理：提交揽收商，统计实际交运数量。

**其他功能**：订单包裹单号录入（无API渠道）、包裹操作人员录入。

### 7.7 供应链域 (SCM) ★ \[AI补货]

**备货分析**：

- 本地仓智能备货分析：根据平台自发货销量、本地库存、采购在途，按规则计算补货建议，一键生成本地仓备货计划单。
- FBA补货建议：支持多种业务场景，自定义日均销量、备货时效、销量去噪规则；计算建议采购量、本地发FBA量、海外仓发FBA量、发货日；加入购物车确认后一键生成FBA货件计划单。
- 海外仓备货分析：根据全平台销量、海外仓库存、本地仓库存、在途数量，按备货规则计算补货数量，一键生成海外仓备库计划单。
- PMS AI补货建议：接收PMS智能补货建议，结合ERP库存数据生成备货计划。

**采购管理**：

- 生成采购单：汇总客户订单/备库订单总需求，减去可用库存和采购中数量，准确计算需采购量，一键生成采购计划单。
- 采购模式：市场采购、工厂采购、天猫淘宝网络采购、1688采购、采购原料自主加工。
- 1688采购：真实在系统内操作1688下单付款；支持自动下单和批量付款。
- 成品加工管理：原料采购、加工、成品入库流程。
- 采购审核流程：按采购类型设置多级下单和付款审核条件，支持自动审核。
- 下单管理：审核采购单，打印采购合同（模板可自定义）。
- 供应商平台：供应商线上接单、回复备货、打印采购条码。
- 下单产品跟单：可视化跟单，汇总下单、收货、质检、入库、不良品退回数量。
- 销单入库核对：查看每次销单质检入库详情及时间。

**平台特殊模式**：

- 支持Temu、Shein、Amazon供应商模式下的备货单、采购、库存管理。

### 7.8 仓储域 (WMS) ★ \[AI预测]

**仓库基础信息**：

- 仓库管理：添加仓库、基本信息。
- 仓库区域/库位管理：按类别、物流属性分区，为产品分配库位。
- 库位分组统计：统计各型号库位占用比例，优化空间利用。
- 所有仓库库存：统一展示本地、海外、FBA多仓库存数量及库位。
- FBA库存管理：查看EFN/NARF库存和库龄。
- Removal订单：FBA退回/销毁/清货处理。
- 自定义标签模板：产品条码、采购条码、拣货单模板、包裹发票等。

**库存事务账**：

- 每次库存变化基于业务来源记录流水，包含在手、预占、可售、在途、不良库存五类状态。
- 在手(on\_hand)：仓库实际持有，入库确认后增加。
- 预占(reserved)：已分配未出库，订单分配后预占。
- 可售(available)：可供销售，在手-预占。
- 在途(in\_transit)：运输中，调拨在途、FBA在途。
- 不良(damaged)：不可销售，质检不良品。

**收货与质检入库**：

- 收货管理：记录收货及异常。
- 销单管理：根据收货数据销单或扫描快递单号快速销单，打印进货单和采购下单条码。
- 质检管理：PC端或PDA扫描条码质检，自动显示质检标准，记录良品/不良品及原因，生成质检报告。
- 不良品管理：查看不良品数量、退货供应商、返货情况。
- 不良品退货管理：按采购回复处理（入库、作废、退货），线上流程化管理。
- 产品返修管理：入库后发现不良品申请返修，出库、维修、重新质检入库全流程。
- 供应商回购/换货：滞销品退货退款，仅换货不退款处理。
- 出货质检管理：拣货后打包前再检，记录发货前不良品。

**仓库其他作业**：

- 手工出入库：办公借用、美工拍照、拍卖、销毁、盘盈入库等场景。
- 调拨管理：多仓库间库存调拨，跟踪在途和收货。
- 盘点管理：支持按整仓、库位、SKU灵活盘点；PDA移动盘点。

### 7.9 FBA/海外仓域 (FBA)

**备货计划**：

- 备库订单新界面统一管理所有仓库备货计划单，也可手动创建。
- 支持修改发货数量、拆分、有货先发、作废计划单。
- FBA补货建议：自定义日均销量、备货时效、销量去噪规则；计算建议采购量、本地发FBA量、海外仓发FBA量、发货日。
- 备货购物车：加入备货购物车确认后一键生成FBA货件计划单。

**发货执行**：

- 调库配货：针对备库订单锁定库存生成包裹，支持自动规则。
- 备库订单包裹：打印FBA产品条码/海外仓产品条码、箱唛、上传装箱清单，流程精细化管理。
- 发货打包SOP：扫描SKU查看特殊包装要求。
- 称重发货：根据装箱清单自动计算重量体积、估算运费；支持电子秤传输、智能设备自动称重量体积。
- 运输统计/交运批次：同客户订单流程。
- 同步海外仓：海外仓包裹发货后手动或自动同步备货计划到海外仓系统。

**FBA备货计划创建模式**：

- 同步Amazon后台：从Amazon后台同步已有货件计划。
- 加载shipmentID：输入Amazon ShipmentID加载货件信息。
- 先创建无ID计划：先在ERP创建计划，后续关联Amazon ShipmentID。

**FBA头程异常处理**：

- 对接Amazon运输中以及仓内破损/丢失/退货/移除/共享库存五大异常场景。
- 自动生成头程批次，关联财务成本归集。

**平台特殊模式**：

- 支持Temu、Shein、Amazon供应商模式下的备货单、采购、库存管理。

### 7.10 物流域 (TMS)

- 异常冲账管理：多发/多采/涨价等异常场景，由采购申请，多级审核后完结冲账。

## 8. 数据需求与治理

### 8.1 核心数据原则

- **SKU分层**：内部SKU唯一，渠道SKU/Listing分离映射。
- **订单分层**：渠道订单（保留原文）、ERP销售单、履约单/包裹单三层分离。
- **库存事务化**：每次库存变化基于业务来源记录流水，包含在手、预占、可售、在途、不良库存。
- **成本事件化**：至少沉淀采购成本、头程、尾程、平台佣金、广告费、支付手续费、仓储费、退款损耗、汇兑损益、税费等成本事件。
- **原始数据保留**：平台订单原文、回调原文、账单原文、物流接口响应原文必须长期保留。
- **数据不可覆盖**：财务相关数据默认不物理删除，原始账单和成本事件不可覆盖。
- **汇率快照化**：汇率必须按生效时间管理（快照化）。
- **结算可追溯**：结算结果必须可追溯到原始来源（订单/采购单/物流单/售后单）。

### 8.2 主数据

- 组织、部门、岗位、用户、角色、权限
- 商品SPU/SKU、品牌、类目、属性、HS Code、申报信息
- 渠道、店铺、市场/站点
- 仓库、库位
- 供应商、物流商、物流渠道
- 汇率、费用项、币种

### 8.3 编码规则

- 内部SKU编码、采购单号(PO)、订单/销售单号(SO)、履约单号(FO)、包裹单号(PK)、售后单号(AF)、结算单号(ST)等需统一规范。

### 8.4 数据库设计要求

- 14域数据库严格隔离，每个域对应独立Schema。
- 所有表包含tenant\_id字段，支持多租户隔离。
- 所有表包含created\_at/updated\_at字段，支持审计追踪。
- 使用JSONB字段存储扩展属性，保持Schema灵活性。
- 使用UUID作为主键，避免ID冲突。

## 13. AI能力与PMS集成需求

### 13.1 AI能力分布原则

```
PMS = 跨域AI决策中心（市场采集、AI分析、选品推荐、跨域经营建议、智能决策辅助）
ERP各域 = 领域规则 + 执行 + 审批 + 反馈
```

ERP域内AI可以保留轻量规则引擎、局部助手或简单模型，但跨域智能推荐由PMS统一编排。

### 13.3 AI交互闭环

```
ERP经营数据 + 外部市场数据
        ↓
PMS AI分析 / 选品 / 定价 / 补货 / 广告 / 风险建议
        ↓
ERP各域接收建议或草稿（非正式业务单据）
        ↓
ERP权限校验 / 规则校验 / 审批流
        ↓
ERP执行正式业务动作
        ↓
执行结果 / 销售效果 / 库存变化 / 利润反馈
        ↓
PMS持续学习优化
```

### 13.4 关键约束

- PMS不直接写入ERP正式业务单据，只写入建议、草稿或待审批动作。
- 所有AI自动执行必须支持开关和灰度，默认需人工审批。
- ERP是经营数据真相源，PMS是AI决策与建议系统，不能反客为主。
- PMS调用必须携带：tenant\_id、actor\_type、actor\_id、scope、purpose、trace\_id、idempotency\_key。
- 审批主控原则：ERP对业务数据拥有完全控制权。
